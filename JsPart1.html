🖥️ Using the Console

The console is a developer tool that allows you to interact with JavaScript code and debug applications.

📝 Note: The console is your best friend for debugging! Use it to test code snippets and check variable values.

Common Console Methods:

// Basic output console.log("Hello World!"); // Different types of messages console.info("This is information"); console.warn("This is a warning"); console.error("This is an error"); // Grouping console messages console.group("User Details"); console.log("Name: John"); console.log("Age: 25"); console.groupEnd(); // Timing operations console.time("Operation Time"); // ... some code console.timeEnd("Operation Time");
🎯 Example: Try opening your browser's developer tools (F12) and run these console commands!

📦 What is a Variable?

A variable is a container that stores data values. Think of it as a labeled box where you can put different items.

📝 Note: Variables are the foundation of programming - they allow us to store and manipulate data!
// Variable declaration and assignment let name = "Alice"; let age = 25; let isStudent = true; console.log(name); // Output: Alice console.log(age); // Output: 25
🎯 Example: Variables are like containers - you can change what's inside them!

🏷️ Data Types in JavaScript

JavaScript has several built-in data types that determine what kind of data can be stored in variables.

Primitive Data Types:

// Number let score = 95; let price = 19.99; // String let message = "Hello World"; let char = 'A'; // Boolean let isActive = true; let isComplete = false; // Undefined let undefinedVar; console.log(undefinedVar); // undefined // Null let emptyValue = null; // Symbol (ES6) let sym = Symbol('id'); // BigInt (ES2020) let bigNumber = 123456789012345678901234567890n;
📝 Note: JavaScript is dynamically typed - you don't need to specify the data type when declaring variables!

Non-Primitive Data Types:

// Object let person = { name: "John", age: 30, city: "New York" }; // Array let colors = ["red", "green", "blue"]; // Function let greet = function(name) { return "Hello " + name; };

🔢 Numbers in JavaScript

JavaScript has only one number type - it can represent both integers and floating-point numbers.

// Integer numbers let age = 25; let temperature = -10; // Floating-point numbers let pi = 3.14159; let price = 99.99; // Scientific notation let bigNumber = 1.5e6; // 1,500,000 let smallNumber = 2.5e-3; // 0.0025 // Special number values let infinity = Infinity; let negativeInfinity = -Infinity; let notANumber = NaN; // Number methods console.log(Number.isInteger(25)); // true console.log(Number.isFinite(Infinity)); // false console.log(Number.parseFloat("3.14")); // 3.14
📝 Note: JavaScript numbers are stored as 64-bit floating-point values, which can sometimes lead to precision issues!
🎯 Example: 0.1 + 0.2 === 0.3 returns false due to floating-point precision!

➕ Operations in JavaScript

JavaScript supports various mathematical and logical operations.

Arithmetic Operators:

let a = 10, b = 3; // Basic arithmetic console.log(a + b); // Addition: 13 console.log(a - b); // Subtraction: 7 console.log(a * b); // Multiplication: 30 console.log(a / b); // Division: 3.333... console.log(a % b); // Modulus: 1 console.log(a ** b); // Exponentiation: 1000 // String concatenation let firstName = "John"; let lastName = "Doe"; console.log(firstName + " " + lastName); // "John Doe"

Comparison Operators:

let x = 5, y = "5"; console.log(x == y); // true (loose equality) console.log(x === y); // false (strict equality) console.log(x != y); // false console.log(x !== y); // true console.log(x > 3); // true console.log(x <= 5); // true

Logical Operators:

let isAdult = true; let hasLicense = false; console.log(isAdult && hasLicense); // AND: false console.log(isAdult || hasLicense); // OR: true console.log(!isAdult); // NOT: false
📝 Note: Always use === for comparison to avoid type coercion surprises!

❌ NaN in JavaScript

NaN stands for "Not a Number" and represents an invalid number value.

// Operations that result in NaN console.log(0 / 0); // NaN console.log("hello" * 2); // NaN console.log(Math.sqrt(-1)); // NaN // Checking for NaN let result = "abc" / 2; console.log(isNaN(result)); // true console.log(Number.isNaN(result)); // true // NaN is not equal to itself! console.log(NaN === NaN); // false console.log(NaN == NaN); // false // Safe way to check for NaN function isReallyNaN(value) { return value !== value; } console.log(isReallyNaN(NaN)); // true
📝 Note: NaN is the only value in JavaScript that is not equal to itself! Use Number.isNaN() for reliable NaN checking.
🎯 Example: parseInt("Hello123") returns NaN because it can't convert the string to a number.

📊 Operator Precedence

Operator precedence determines the order in which operations are performed in expressions.

// Precedence examples console.log(2 + 3 * 4); // 14 (not 20) console.log((2 + 3) * 4); // 20 // Common precedence order (highest to lowest) // 1. Parentheses () // 2. Exponentiation ** // 3. Multiplication *, Division /, Modulus % // 4. Addition +, Subtraction - // 5. Comparison operators <, >, <=, >= // 6. Equality ==, ===, !=, !== // 7. Logical AND && // 8. Logical OR || // 9. Assignment =, +=, -=, etc. let result = 10 + 5 * 2 ** 3 / 4 - 1; console.log(result); // 19 (10 + 5 * 8 / 4 - 1 = 10 + 10 - 1)
📝 Note: When in doubt, use parentheses to make your intentions clear!

🔑 let, const & var Keywords

These keywords are used to declare variables, each with different behaviors and scope rules.

var (Old way - avoid in modern JS):

var name = "John"; var name = "Jane"; // Redeclaration allowed name = "Bob"; // Reassignment allowed // Function-scoped function example() { var x = 1; if (true) { var x = 2; // Same variable console.log(x); // 2 } console.log(x); // 2 }

let (Block-scoped, reassignable):

let age = 25; // let age = 30; // Error: Cannot redeclare age = 30; // Reassignment allowed // Block-scoped function example() { let y = 1; if (true) { let y = 2; // Different variable console.log(y); // 2 } console.log(y); // 1 }

const (Block-scoped, not reassignable):

const PI = 3.14159; // PI = 3.14; // Error: Cannot reassign // Objects and arrays can be mutated const person = { name: "John" }; person.name = "Jane"; // Allowed person.age = 25; // Allowed const colors = ["red", "green"]; colors.push("blue"); // Allowed colors[0] = "yellow"; // Allowed
📝 Note: Use const by default, let when you need to reassign, and avoid var in modern JavaScript!

= Assignment Operators

Assignment operators are used to assign values to variables.

let x = 10; // Basic assignment x = 20; // Compound assignment operators x += 5; // x = x + 5 (25) x -= 3; // x = x - 3 (22) x *= 2; // x = x * 2 (44) x /= 4; // x = x / 4 (11) x %= 3; // x = x % 3 (2) x **= 3; // x = x ** 3 (8) // Logical assignment (ES2021) let a = null; a ||= "default"; // a = a || "default" console.log(a); // "default" let b = "hello"; b &&= "world"; // b = b && "world" console.log(b); // "world" let c = null; c ??= "fallback"; // c = c ?? "fallback" console.log(c); // "fallback"
🎯 Example: Compound assignment operators are shorthand for common operations!

1️⃣ Unary Operators

Unary operators work with a single operand.

let x = 5; let y = "10"; // Increment and Decrement console.log(++x); // Pre-increment: 6 console.log(x++); // Post-increment: 6 (then x becomes 7) console.log(--x); // Pre-decrement: 6 console.log(x--); // Post-decrement: 6 (then x becomes 5) // Unary plus and minus console.log(+y); // 10 (converts string to number) console.log(-x); // -5 // typeof operator console.log(typeof x); // "number" console.log(typeof "hi"); // "string" console.log(typeof true); // "boolean" console.log(typeof {}); // "object" // Logical NOT console.log(!true); // false console.log(!!x); // true (double negation converts to boolean) // delete operator let obj = { name: "John", age: 30 }; delete obj.age; console.log(obj); // { name: "John" }
📝 Note: Be careful with pre/post increment operators - they can cause confusion in complex expressions!

🏷️ Identifier Rules

Identifiers are names used for variables, functions, and objects. They must follow specific rules.

// Valid identifiers let name; let firstName; let _private; let $jquery; let userName123; let camelCase; let CONSTANT_VALUE; // Invalid identifiers // let 123name; // Cannot start with number // let first-name; // Cannot contain hyphens // let first name; // Cannot contain spaces // let class; // Cannot use reserved words // let function; // Cannot use reserved words // Reserved words (cannot be used as identifiers) const reservedWords = [ 'break', 'case', 'catch', 'class', 'const', 'continue', 'debugger', 'default', 'delete', 'do', 'else', 'export', 'extends', 'finally', 'for', 'function', 'if', 'import', 'in', 'instanceof', 'let', 'new', 'return', 'super', 'switch', 'this', 'throw', 'try', 'typeof', 'var', 'void', 'while', 'with', 'yield' ];
📝 Rules:
• Must start with letter, underscore (_), or dollar sign ($)
• Can contain letters, numbers, underscores, and dollar signs
• Case-sensitive (myVar and MyVar are different)
• Cannot be reserved keywords
• Use camelCase convention for variables and functions

✅ Boolean in JavaScript

Boolean represents logical values: true or false.

// Boolean literals let isTrue = true; let isFalse = false; // Boolean conversion console.log(Boolean(1)); // true console.log(Boolean(0)); // false console.log(Boolean("hello")); // true console.log(Boolean("")); // false console.log(Boolean(null)); // false console.log(Boolean(undefined)); // false // Truthy values if ("hello") console.log("Truthy!"); // Executes if (42) console.log("Truthy!"); // Executes if ([]) console.log("Truthy!"); // Executes (empty array is truthy) if ({}) console.log("Truthy!"); // Executes (empty object is truthy) // Falsy values const falsyValues = [false, 0, -0, 0n, "", null, undefined, NaN]; // Boolean operations let a = true, b = false; console.log(a && b); // false console.log(a || b); // true console.log(!a); // false // Short-circuit evaluation let name = null; let displayName = name || "Guest"; // "Guest" console.log(displayName);
📝 Note: Remember the 8 falsy values: false, 0, -0, 0n, "", null, undefined, NaN. Everything else is truthy!

🔷 What is TypeScript?

TypeScript is a statically typed superset of JavaScript that compiles to plain JavaScript.

// JavaScript (dynamic typing) let message = "Hello"; message = 42; // Valid in JS // TypeScript (static typing) let message: string = "Hello"; // message = 42; // Error in TS // Type annotations let age: number = 25; let isStudent: boolean = true; let hobbies: string[] = ["reading", "coding"]; // Interface interface Person { name: string; age: number; email?: string; // Optional property } let user: Person = { name: "John", age: 30 }; // Function with types function greet(name: string): string { return `Hello, ${name}!`; } // Generic types function identity(arg: T): T { return arg; }
📝 Benefits of TypeScript:
• Static type checking catches errors at compile time
• Better IDE support with autocomplete and refactoring
• Enhanced code documentation and readability
• Easier maintenance of large codebases
• Compiles to clean, readable JavaScript
🎯 Example: TypeScript helps prevent common bugs like accessing properties on null/undefined values!

📝 String in JavaScript

Strings represent text data and are immutable in JavaScript.

// String creation let singleQuotes = 'Hello World'; let doubleQuotes = "Hello World"; let templateLiteral = `Hello World`; // String with variables (template literals) let name = "Alice"; let greeting = `Hello, ${name}! Today is ${new Date().toDateString()}`; // Multiline strings let multiline = ` This is a multiline string `; // String methods let text = "JavaScript Programming"; console.log(text.length); // 21 console.log(text.toUpperCase()); // "JAVASCRIPT PROGRAMMING" console.log(text.toLowerCase()); // "javascript programming" console.log(text.charAt(0)); // "J" console.log(text.substring(0, 10)); // "JavaScript" console.log(text.indexOf("Script")); // 4 console.log(text.includes("Program")); // true console.log(text.split(" ")); // ["JavaScript", "Programming"] // String immutability let original = "Hello"; original.toUpperCase(); // Returns "HELLO" but doesn't change original console.log(original); // Still "Hello"
📝 Note: Strings are immutable - methods that appear to modify strings actually return new strings!

🔤 String Indices

String characters can be accessed using indices, starting from 0.

let word = "JavaScript"; // Accessing characters by index console.log(word[0]); // "J" console.log(word[1]); // "a" console.log(word[4]); // "S" console.log(word[9]); // "t" // Last character console.log(word[word.length - 1]); // "t" // Negative indices (doesn't work directly) // console.log(word[-1]); // undefined // Using charAt method console.log(word.charAt(0)); // "J" console.log(word.charAt(50)); // "" (empty string, not undefined) // Looping through string for (let i = 0; i < word.length; i++) { console.log(`Index ${i}: ${word[i]}`); } // Modern way with for...of for (let char of word) { console.log(char); } // String is array-like but not an array console.log(Array.isArray(word)); // false console.log(Array.isArray([...word])); // true
🎯 Example: String indices are like house numbers - they start from 0 and go up to length-1!

🚫 null & undefined in JavaScript

Both represent "nothing" but in different contexts.

// undefined - variable declared but not assigned let notAssigned; console.log(notAssigned); // undefined console.log(typeof notAssigned); // "undefined" // null - intentional absence of value let emptyValue = null; console.log(emptyValue); // null console.log(typeof emptyValue); // "object" (this is a known bug in JS) // Common scenarios for undefined let obj = { name: "John" }; console.log(obj.age); // undefined (property doesn't exist) function greet(name) { console.log(name); // undefined if no argument passed } greet(); // undefined // Common scenarios for null let data = null; // Explicitly set to "no value" let element = document.getElementById("nonexistent"); // null // Comparison console.log(null == undefined); // true (loose equality) console.log(null === undefined); // false (strict equality) // Checking for null/undefined function checkValue(value) { if (value == null) { console.log("Value is null or undefined"); } // More specific checks if (value === null) { console.log("Value is null"); } else if (value === undefined) { console.log("Value is undefined"); } } // Nullish coalescing operator (??) - ES2020 let username = null; let displayName = username ?? "Guest"; // "Guest" let age = 0; let userAge = age ?? 18; // 0 (because 0 is not null/undefined) // Optional chaining (?.) - ES2020 let user = { profile: { name: "John" } }; console.log(user?.profile?.name); // "John" console.log(user?.address?.street); // undefined (no error)
📝 Key Differences:
• undefined: Variable exists but has no value assigned
• null: Intentionally set to represent "no value"
• Use null when you want to explicitly indicate "empty"
• undefined is the default for uninitialized variables
🎯 Example: Think of undefined as "I forgot to put something here" and null as "I intentionally left this empty"!

🎯 Interview Questions

Essential questions for placement preparation

Q1: What is the difference between let, const, and var?

Answer: var is function-scoped and can be redeclared; let is block-scoped and reassignable but not redeclarable; const is block-scoped and neither reassignable nor redeclarable. var has hoisting issues, while let and const have temporal dead zone.

Q2: Explain the difference between == and === operators.

Answer: == performs type coercion and compares values after converting them to the same type. === performs strict comparison without type coercion. Example: "5" == 5 is true, but "5" === 5 is false.

Q3: What are falsy values in JavaScript?

Answer: There are 8 falsy values: false, 0, -0, 0n (BigInt zero), "" (empty string), null, undefined, and NaN. Everything else is truthy.

Q4: What is NaN and how do you check for it?

Answer: NaN means "Not a Number" and represents an invalid number. It's unique because NaN !== NaN. Use Number.isNaN() to check for NaN reliably, as isNaN() can give false positives.

Q5: Explain the difference between null and undefined.

Answer: undefined means a variable is declared but not assigned a value. null is an assignment value representing intentional absence of value. typeof null returns "object" (a known bug), while typeof undefined returns "undefined".

Q6: What is hoisting in JavaScript?

Answer: Hoisting is JavaScript's behavior of moving variable and function declarations to the top of their scope during compilation. var variables are hoisted and initialized with undefined, while let/const are hoisted but not initialized (temporal dead zone).

Q7: What are template literals and their advantages?

Answer: Template literals use backticks (`) and allow embedded expressions with ${}, multiline strings, and tagged templates. They're more readable than string concatenation and support expression interpolation.

Q8: Explain operator precedence with an example.

Answer: Operator precedence determines evaluation order. Multiplication has higher precedence than addition. Example: 2 + 3 * 4 = 14 (not 20) because 3 * 4 is evaluated first.

Q9: What is the difference between pre-increment and post-increment?

Answer: Pre-increment (++x) increments the value and returns the new value. Post-increment (x++) returns the current value and then increments. Example: if x=5, ++x returns 6 and x becomes 6; x++ returns 5 and x becomes 6.

Q10: How do you check the data type of a variable?

Answer: Use the typeof operator. Example: typeof "hello" returns "string". Note that typeof null returns "object" (historical bug), and typeof [] returns "object". Use Array.isArray() to check for arrays specifically.

Q11: What are the rules for JavaScript identifiers?

Answer: Identifiers must start with a letter, underscore, or dollar sign. Subsequent characters can include numbers. They're case-sensitive, cannot be reserved words, and should follow camelCase convention.

Q12: Explain string immutability in JavaScript.

Answer: Strings are immutable, meaning they cannot be changed after creation. String methods return new strings rather than modifying the original. Example: str.toUpperCase() returns a new string; it doesn't change str.

Q13: What is TypeScript and why use it?

Answer: TypeScript is a statically typed superset of JavaScript that compiles to plain JavaScript. Benefits include early error detection, better IDE support, enhanced code documentation, and easier maintenance of large applications.

Q14: What is the nullish coalescing operator?

Answer: The ?? operator returns the right operand when the left is null or undefined, but not for other falsy values. Example: 0 ?? "default" returns 0, while null ?? "default" returns "default".

Q15: Explain the console object and its methods.

Answer: The console object provides access to debugging console. Methods include log() for general output, error() for errors, warn() for warnings, time()/timeEnd() for timing, and group()/groupEnd() for organizing output.
