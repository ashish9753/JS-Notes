üñ•Ô∏è console.log()

The console.log() method is used to print/display messages to the browser's console. It's essential for debugging and testing JavaScript code.
üí° Examples:
// Basic console.log console.log("Hello World!"); // Multiple values console.log("Name:", "John", "Age:", 25); // Variables let name = "Alice"; console.log("User name is:", name); // Objects and Arrays console.log({name: "Bob", age: 30}); console.log([1, 2, 3, 4]);
üìå Important Note:
console.log() only displays output in the browser's developer console (F12), not on the webpage itself.

üîó Linking JS File

There are multiple ways to include JavaScript in your HTML document: inline, internal, and external linking.
üí° Methods to Link JavaScript:
<script src="script.js"></script> <script> console.log("Internal JS"); </script> <button onclick="alert('Hello!')">Click Me</button>
üìå Best Practice:
Always place script tags before the closing </body> tag for better performance, or use external files for better organization.

üìù Template Literals

Template literals use backticks (`) instead of quotes and allow embedded expressions using ${expression} syntax. They support multi-line strings and string interpolation.
üí° Examples:
// Basic template literal let name = "John"; let greeting = `Hello, ${name}!`; console.log(greeting); // Hello, John! // Multi-line strings let multiLine = `This is line 1 This is line 2 This is line 3`; // Expressions in template literals let a = 5, b = 10; console.log(`Sum: ${a + b}`); // Sum: 15 // Function calls console.log(`Current time: ${new Date().getHours()}:${new Date().getMinutes()}`);
üìå Key Advantage:
Template literals make string concatenation much cleaner and more readable compared to traditional string concatenation with + operator.

‚ö° Operators in JS

JavaScript operators are symbols used to perform operations on operands (variables and values). There are several types of operators.
üí° Types of Operators:
// Arithmetic Operators let x = 10, y = 3; console.log(x + y); // Addition: 13 console.log(x - y); // Subtraction: 7 console.log(x * y); // Multiplication: 30 console.log(x / y); // Division: 3.333... console.log(x % y); // Modulus: 1 console.log(x ** y); // Exponentiation: 1000 // Assignment Operators let num = 5; num += 3; // num = num + 3 = 8 num -= 2; // num = num - 2 = 6 num *= 2; // num = num * 2 = 12 // Increment/Decrement let count = 5; count++; // count = 6 (post-increment) ++count; // count = 7 (pre-increment) count--; // count = 6 (post-decrement)

üîç Comparison Operators

Comparison operators compare two values and return a boolean result (true or false). They are essential for conditional statements and loops.
üí° All Comparison Operators:
let a = 5, b = 10, c = "5"; // Equal to (loose equality) console.log(a == 5); // true console.log(a == c); // true (type coercion) // Strict equal to (strict equality) console.log(a === 5); // true console.log(a === c); // false (different types) // Not equal to console.log(a != b); // true console.log(a !== c); // true (strict inequality) // Greater than / Less than console.log(b > a); // true console.log(a < b); // true console.log(a >= 5); // true console.log(a <= 4); // false
üìå Important Difference:
Always use === (strict equality) instead of == to avoid unexpected type coercion issues.

üî¢ Comparison for Non-numbers

When comparing non-numeric values, JavaScript follows specific rules for strings, booleans, and other data types.
üí° String Comparisons:
// String comparison (lexicographic order) console.log("apple" < "banana"); // true console.log("Apple" < "apple"); // true (uppercase comes first) console.log("10" < "2"); // true (string comparison, not numeric) // Boolean comparisons console.log(true == 1); // true console.log(false == 0); // true console.log(true === 1); // false // Mixed type comparisons console.log("5" > 3); // true (string "5" converted to number) console.log(null == undefined); // true console.log(null === undefined);// false

ü§î Conditional Statements

Conditional statements allow you to execute different blocks of code based on different conditions. They are the foundation of decision-making in programming.

‚úÖ if Statement

The if statement executes a block of code if a specified condition is true.
üí° Examples:
// Basic if statement let age = 18; if (age >= 18) { console.log("You are an adult!"); } // if with multiple conditions let score = 85; if (score >= 80 && score <= 100) { console.log("Excellent grade!"); } // if without braces (single statement) if (age < 13) console.log("You are a child");

üîÑ else if Statement

The else if statement allows you to test multiple conditions in sequence. It's used when you have more than two possible outcomes.
üí° Examples:
// Grade classification let marks = 75; if (marks >= 90) { console.log("Grade A"); } else if (marks >= 80) { console.log("Grade B"); } else if (marks >= 70) { console.log("Grade C"); } else if (marks >= 60) { console.log("Grade D"); } // Time-based greeting let hour = new Date().getHours(); if (hour < 12) { console.log("Good Morning!"); } else if (hour < 17) { console.log("Good Afternoon!"); } else if (hour < 21) { console.log("Good Evening!"); }

‚ùå else Statement

The else statement executes a block of code when all preceding if and else if conditions are false. It's the final fallback option.
üí° Complete if-else chain:
let weather = "rainy"; if (weather === "sunny") { console.log("Let's go to the beach!"); } else if (weather === "cloudy") { console.log("Perfect for a walk!"); } else if (weather === "rainy") { console.log("Stay inside and read a book!"); } else { console.log("I don't know this weather type!"); } // Simple if-else let number = 7; if (number % 2 === 0) { console.log("Even number"); } else { console.log("Odd number"); }

üîó Nested if-else

Nested if-else statements are if statements inside other if statements. They allow for more complex decision-making with multiple levels of conditions.
üí° Examples:
// Age and license check let age = 20; let hasLicense = true; if (age >= 18) { if (hasLicense) { console.log("You can drive!"); } else { console.log("You need to get a license first."); } } else { console.log("You're too young to drive."); } // Grade with attendance check let score = 85; let attendance = 90; if (score >= 80) { if (attendance >= 85) { console.log("Excellent performance!"); } else { console.log("Good score but improve attendance."); } } else { if (attendance >= 85) { console.log("Good attendance but improve scores."); } else { console.log("Need improvement in both areas."); } }

‚ú® Logical Operators

Logical operators are used to combine or modify boolean values. They're essential for creating complex conditions in if statements and loops.
üí° The Three Logical Operators:
// AND (&&) - Both conditions must be true let age = 25; let hasJob = true; if (age >= 18 && hasJob) { console.log("Eligible for loan"); } // OR (||) - At least one condition must be true let isWeekend = true; let isHoliday = false; if (isWeekend || isHoliday) { console.log("No work today!"); } // NOT (!) - Reverses the boolean value let isLoggedIn = false; if (!isLoggedIn) { console.log("Please log in first"); } // Complex combinations let score = 75; let attendance = 90; let hasBonus = false; if ((score >= 70 && attendance >= 85) || hasBonus) { console.log("Promotion eligible!"); } // Short-circuit evaluation let user = {name: "John"}; user && user.name && console.log("Hello, " + user.name);
üìå Operator Precedence:
Remember: ! (NOT) has higher precedence than && (AND), which has higher precedence than || (OR). Use parentheses for clarity.

üéØ Truthy & Falsy

In JavaScript, every value has an inherent boolean value. Understanding truthy and falsy values is crucial for conditional statements and logical operations.
üí° Falsy Values (only 6 in JavaScript):
// The 6 falsy values in JavaScript console.log(Boolean(false)); // false console.log(Boolean(0)); // false console.log(Boolean(-0)); // false console.log(Boolean("")); // false (empty string) console.log(Boolean(null)); // false console.log(Boolean(undefined)); // false console.log(Boolean(NaN)); // false // Practical examples let name = ""; if (!name) { console.log("Name is required!"); } let age = 0; if (!age) { console.log("Age not provided"); }
üí° Truthy Values (everything else):
// All other values are truthy console.log(Boolean(true)); // true console.log(Boolean(1)); // true console.log(Boolean(-1)); // true console.log(Boolean("hello")); // true console.log(Boolean(" ")); // true (space is not empty) console.log(Boolean([])); // true (empty array) console.log(Boolean({})); // true (empty object) console.log(Boolean(function(){})); // true // Common use cases let user = {name: "Alice"}; if (user) { console.log("User exists"); } let items = [1, 2, 3]; if (items.length) { console.log("Items available"); }
üìå Important Note:
Be careful with "0", "false", "null" as strings - they are truthy! Only the actual values are falsy.

üîÄ Switch Statement

The switch statement is used to execute different blocks of code based on different values. It's often cleaner than multiple if-else statements when comparing the same variable against many values.
üí° Basic Switch Statement:
// Day of the week example let day = 3; let dayName; switch (day) { case 1: dayName = "Monday"; break; case 2: dayName = "Tuesday"; break; case 3: dayName = "Wednesday"; break; case 4: dayName = "Thursday"; break; case 5: dayName = "Friday"; break; default: dayName = "Weekend"; } console.log(dayName); // Wednesday
üí° Advanced Switch Examples:
// Multiple cases with same result let grade = "B"; switch (grade) { case "A": case "A+": console.log("Excellent!"); break; case "B": case "B+": console.log("Good job!"); break; case "C": console.log("Average"); break; default: console.log("Need improvement"); } // Calculator example let num1 = 10, num2 = 5, operator = "+"; let result; switch (operator) { case "+": result = num1 + num2; break; case "-": result = num1 - num2; break; case "*": result = num1 * num2; break; case "/": result = num2 !== 0 ? num1 / num2 : "Cannot divide by zero"; break; default: result = "Invalid operator"; } console.log(`Result: ${result}`);
üìå Important:
Always use 'break' statements to prevent fall-through behavior. The default case is optional but recommended.

üéØ Interview Questions & Answers

Q1: What's the difference between == and === operators?
Answer: == (loose equality) performs type coercion and compares values after converting them to the same type, while === (strict equality) compares both value and type without any conversion. Example: '5' == 5 is true, but '5' === 5 is false.
Q2: What are the falsy values in JavaScript?
Answer: There are exactly 6 falsy values in JavaScript: false, 0, -0, "" (empty string), null, undefined, and NaN. Everything else is truthy.
Q3: Explain the difference between if-else and switch statements.
Answer: if-else is better for complex conditions and ranges, while switch is cleaner for comparing a single variable against multiple specific values. Switch uses strict equality (===) for comparison and requires break statements to prevent fall-through.
Q4: What is short-circuit evaluation in logical operators?
Answer: Short-circuit evaluation means that in logical operations, if the result can be determined from the first operand, the second operand is not evaluated. For &&, if the first is false, the result is false. For ||, if the first is true, the result is true.
Q5: How do template literals differ from regular strings?
Answer: Template literals use backticks (`) and support string interpolation with ${expression}, multi-line strings without escape characters, and embedded expressions. Regular strings use quotes and require concatenation (+) for dynamic content.
Q6: What happens if you omit the break statement in a switch case?
Answer: Without break, execution continues to the next case (fall-through), executing all subsequent cases until a break is encountered or the switch ends. This is usually unintended behavior.
Q7: How does JavaScript handle string comparison?
Answer: JavaScript compares strings lexicographically (dictionary order) character by character using Unicode values. Uppercase letters come before lowercase letters. For example: "Apple" < "apple" is true, and "10" < "2" is true (string comparison, not numeric).
Q8: What's the difference between alert(), prompt(), and confirm()?
Answer: alert() displays a message and returns undefined. prompt() displays a message with an input field and returns the user's input as a string (or null if cancelled). confirm() displays a message with OK/Cancel buttons and returns true/false.
Q9: When should you use nested if-else vs logical operators?
Answer: Use logical operators (&&, ||) for simple conditions that can be evaluated in one expression. Use nested if-else when you need different actions for different condition combinations or when the logic is too complex for operators.
Q10: How do you check if a variable is undefined or null?
Answer: You can use: if (variable == null) to check for both null and undefined, or if (variable === null || variable === undefined) for explicit checking, or if (typeof variable === 'undefined') specifically for undefined.
Q11: What's the output of: console.log(0 || 5 || 10)?
Answer: The output is 5. The || operator returns the first truthy value it encounters. Since 0 is falsy, it moves to 5, which is truthy, so 5 is returned without evaluating 10.
Q12: How do you link an external JavaScript file to HTML?
Answer: Use the script tag with src attribute: <script src="filename.js"></script>. Best practice is to place it before the closing </body> tag for better performance, or use defer/async attributes in the head.
